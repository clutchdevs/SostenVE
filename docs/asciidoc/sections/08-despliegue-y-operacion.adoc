==== Fuente vs. artefacto de runtime

(como `.ts` â†’ `.js` o SCSS â†’ CSS), *no* una duplicaciĂ³n:

|===
| Archivo | Rol | Â¿Se edita a mano? | Â¿Lo carga el runtime?
| `apps/api/config/app.config.yml` | *Fuente de verdad* (legible, con comentarios y anclas por entorno) | *SĂ­ â€” solo este* | No
| `apps/api/src/config/app-config.generated.ts` | *Artefacto* horneado por el build | Nunca | *SĂ­ â€” este*

*Por quĂ© existen los dos (y por quĂ© se commitean ambos):* en Vercel serverless *no se pueden leer

estĂ©n en sync. Borrar el YAML no romperĂ­a la ejecuciĂ³n (usa el generado), pero se perderĂ­a la fuente
editable: por eso el YAML es *imprescindible* y el generado nunca se edita a mano.

. Se ejecuta `npm run config:build` (`scripts/build-config.mjs`), que "hornea" el YAML en un mĂ³dulo
